С чего начать изучать программирование?

PYTHON

Python—высокоуровневый язык программирования  общего назначения с  динамической строгой  типизацией и автоматическим управлением памятью, ориентированный на повышение производительности разработчика, читаемости кода и его качества, а также на обеспечение переносимости написанных на нём программ.

Гвидо ван Россум

История создания:

Изначально он занимался разработкой языка ABC. Язык ABC разрабатывался в конце 1970-х и начале 1980-х годов в Центре математики Амстердамского университета совместно с французским исследовательским институтом INRIA.

ABC был разработан для обучения программированию и характеризовался простотой и доступностью, но не решал сложные задачи.В связи с этим ABC не получил большой популярности в промышленной разработке, но стал предпосылкой для развития чего-то большего.

В декабре 1989 года Гвидо начал создавать язык, который сочетал бы простоту ABC с более широкими возможностями. Так появился Python.

История развития:

  • В феврале 1991 года Гвидо ван Россум опубликовал код Python, помеченный версией 0.9.0, на alt.sources. На этой стадии в нём уже присутствовали классы с наследованием, обработка исключений, функции и основные типы данных: list, dict, str и т. д.
  • Также в этом начальном релизе были модули. Ван Россум описывал модуль как «один из главных элементов в программировании на Python».
  • В 1994 году с ростом числа пользователей сформировалась группа comp.lang.python — основной форум Python. В этом же году были добавлены элементы функционального программирования и ООП. Изначально пайтон был написан на языке программирования "С", что послужило его довольно быстрому выполнению.
  • Версия Python 2.0 была выпущена 16 октября 2000 года и включала в себя много новых крупных функций — таких как полный сборщик мусора и поддержка Unicode. Однако наиболее важным из всех изменений было изменение самого процесса развития языка и переход на более прозрачный процесс его создания Эта версия сыграла большую роль в популизации Python, поскольку была одобрена и принята множеством разработчиков.
  • Первая обратно-несовместимая версия Python 3.0 была выпущена 3 декабря 2008 года после длительного периода тестирования. Основной проблемой версии 2.0 и последующих ей стало то, что накопилось множество способов решения одних и тех же задач, что противоречило принципу: "Должен существовать один и, желательно, только один очевидный способ сделать это."

Решаемые задачи

Арифметические:

  • Вычисление суммы и произведения цифр числа. Задача осложняется тем, что количество разрядов числа заранее не известно. Решение — последовательные арифметические действия: извлечь из числа все цифры, сложить их и перемножить. В Python используются операции деления нацело и нахождения остатка.

  • Вычисление процента по банковскому вкладу. Пользователь вводит начальную сумму, процент по вкладу и количество лет, а программа выводит сумму начисленных процентов. Решение — формула (P × R × T) / 100, где P — начальная сумма, R — процент по вкладу, T — количество лет.

Задачи со списками:

  • Подсчёт количества элементов в списке. Решение — написать программу, которая считает количество элементов в заданном списке и выводит это количество на экран.

  • Обратный порядок элементов списка. Решение — написать программу, которая принимает список и выводит его элементы в обратном порядке.

  • Удаление повторяющихся элементов из списка. Решение — написать программу, которая удаляет все повторяющиеся элементы из заданного списка и выводит список без повторов.

Создание словаря:

  • Создание словаря, в котором ключами будут числа от 1 до 10, а значениями — эти же числа, возведённые в куб. Решение — использовать цикл for, а для получения чисел в диапазоне от 1 до 10 — функцию range().

  • Создание словаря из строки: в качестве ключей — буквы строки, а значениями — числа, соответствующие количеству вхождений данной буквы в строку. Решение — использовать функцию count() для генерации словаря.

Java Script

JavaScript —мультипарадигменный язык программирования. Поддерживает объектно-ориентированный, императивный и функциональный стили. Является реализацией спецификации ECMAScript. JavaScript обычно используется как встраиваемый язык для программного доступа к объектам приложений. Наиболее широкое применение находит в браузерах как язык сценариев для придания интерактивности веб-страницам.

Официальная документация:

клик

Брендан Эйх

История создания:

Перед Бренданом Эйхом была поставлена задача внедрить язык программирования Scheme или что-то похожее в браузер Netscape.

В мае он немедленно начал разрабатывать концепцию нового языка программирования. Менеджмент разработки браузера, включая Тома Пакина, Михаэля Тоя, Рика Шелла, был убеждён, что Netscape должен поддерживать язык программирования, встраиваемый в HTML-код страницы.

Первоначально по предложению Марка Андрессена язык был назван Mocha, был реализован Бренданом Эйхом в течение десяти дней и впервые был включен в альфу версию Netscape 2. Затем он был переименован в JavaScript. Анонс JavaScript со стороны представителей Netscape и Sun состоялся накануне выпуска второй бета-версии Netscape Navigator.

История развития:

  • В 1997 году JavaScript был стандартизирован ассоциацией ECMA International как ECMAScript (ECMA-262). Стандарт был необходим для обеспечения совместимости между различными реализациями языка в разных браузерах.
  • Название ECMAScript стало компромиссом между организациями, участвовавшими в стандартизации языка, особенно Netscape и Microsoft.
  • В 1994 году с ростом числа пользователей сформировалась группа comp.lang.python — основной форум Python. В этом же году были добавлены элементы функционального программирования и ООП. Изначально пайтон был написан на языке программирования "С", что послужило его довольно быстрому выполнению.
  • Некоторые этапы развития JavaScript:

    - ECMAScript 3 (1999) — обновление, добавившее регулярные выражения, обработку исключений try/catch, улучшенную работу со строками и более строгое определение ошибок.

    - ECMAScript 5 (2009) — переработанный стандарт, добавивший строгий режим («use strict»), поддержку JSON, новые методы массивов (forEach, map, filter, reduce) и улучшенную рефлексию объектов.

    - ECMAScript 2015 (ES6) — версия с революционными нововведениями: классы, модули, стрелочные функции, let/const, деструктуризация, промисы для асинхронного программирования.
  • JavaScript эволюционировал от простого скриптового языка для «оживления» веб-страниц в мощный инструмент для создания сложных веб-приложений, серверных приложений и мобильных приложений.

Решаемые задачи

Алгоритмы:

  • Поиск уникального элемента в массиве. Нужно написать функцию, которая находит уникальный элемент в массиве чисел, где все числа повторяются дважды, за исключением одного. Решение: использовать объект ElementCount для подсчёта количества вхождений каждого элемента в массиве, затем пройти по объекту для поиска элемента с единственным вхождением.

  • Проверка, является ли переданная строка палиндромом. Палиндромом считается строка, которая читается одинаково как слева направо, так и справа налево. Решение: разбить строку на массив символов, изменить порядок элементов массива (сделать его зеркальным) и объединить элементы массива обратно в строку. Затем сравнить исходную строку с её зеркальным отражением: если строки равны, то возвращается true, иначе — false.

Строки:

  • Преобразование строк. Например, написать функцию, которая принимает параметр с строкой и преобразовывает её в следующий вид: «Привет Мир» => «прИвЕт мИр», «Азбука Вкуса» => «АзбУкА ВкУсА». Решение: использовать цикл for, который проходит по всем буквам в строке и проверяет, являются ли они гласными или согласными. Если буква гласная, она попадает в результат и преобразуется в верхний регистр, если согласная — в нижний.

Массивы:

  • Сортировка массивов. Например, написать функцию, которая принимает массив целых чисел и определяет, отсортирован ли он по возрастанию.

  • Разделение массива на части. Нужно написать функцию, которая разделяет массив на части заданного размера.

  • Чётный отсев. Нужно написать функцию, принимающую массив и возвращающую новый, где удалён каждый второй элемент.

Объекты:

  • Проверка, является ли объект пустым. Нужно написать стрелочную функцию, которая проверяет, не содержит ли объект собственные свойства и методы.

  • Сортировка массива объектов. Например, нужно отсортировать массив объектов по полю likes.

  • Создание нового массива. Нужно оставить в нём только те объекты, возраст которых больше или равно 18.
